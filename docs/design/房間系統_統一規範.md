# 房間系統統一規範

> 版本：v1.0 | 日期：2026-05-11
> 適用：甜甜的小秘密 所有需要「建立房間 / 加入房間」的 App

---

## 一、問題背景

目前平台上兩個多人遊戲的房間建立流程**不一致**：

| App | 建立房間方式 | 加入房間方式 |
|-----|-------------|-------------|
| 懂我再說 | 前端直接呼叫 `DB.createRoom()` 寫入 Supabase rooms 表 | Socket `join_room` 驗證房間存在後加入 |
| 默契傳聲筒（v1 草稿） | 前端只產生 roomId，透過 `cs:create` event 讓後端建立（in-memory） | Socket `cs:join` 加入 |

這導致：
- 後端 in-memory 的房間在 **server 重啟後消失**（無法斷線重連）
- 不同遊戲的前端 Lobby 頁無法共用同一套邏輯
- 未來新 App 要各自重新摸索一套流程

---

## 二、統一架構設計

### 2.1 核心原則

```
「房間由後端建立並持久化，前端只做 UI 和路由」
```

1. **Supabase `rooms` 表是唯一的房間儲存源**（所有 App 共用同一張表）
2. **後端 Socket.io** 處理所有加入邏輯（驗證、人數限制、暱稱重複）
3. **前端只負責**：產生 roomId、收集使用者輸入、跳轉路由
4. `shared/data/identity.js` 的 `generateRoomId()` 全平台共用，不重複實作

### 2.2 統一的 Supabase rooms 表

**所有 App 共用此表，以 `app_id` 欄位區分遊戲：**

```sql
-- 在現有 rooms 表新增 app_id 欄（若尚未存在）
ALTER TABLE rooms ADD COLUMN IF NOT EXISTS
  app_id TEXT NOT NULL DEFAULT 'know-me';

-- App 識別值對應
-- 'know-me'         → 懂我再說
-- 'character-storm' → 默契傳聲筒
-- （未來新增 App 只需新增一個值）
```

> ⚠️ `app_id` 的值必須和 `shared/data/tools.js` 中的 `id` 欄位一致。

## 三、兩種建立房間模式說明

### 模式 A：前端直接建立（懂我再說 現況）

```
LobbyPage
  │
  ├─ 使用者點「建立房間」
  ├─ 前端呼叫 DB.createRoom(roomId, playerId, maxPlayers, appId)
  │    → 直接寫入 Supabase rooms 表
  └─ router.push 到 RoomPage（帶 query: id, nickname）
          │
          └─ RoomPage socket.emit('join_room', { roomId, playerId, nickname })
               → 後端驗證房間存在 → 建立玩家 → 推送 room_state
```

**優點**：前端邏輯簡單，不依賴 Socket 連線完成才能建立
**缺點**：前端直接用 anon key 寫入 DB，需要 RLS 配合；後端 restart 後需從 DB reload 快取

### 模式 B：後端透過 Socket 建立（推薦新 App 採用）

```
LobbyPage
  │
  ├─ 使用者點「建立房間」
  ├─ 前端 router.push 到 RoomPage（帶 query: id, nickname, create=1, max=N）
  │    ← 不做任何 DB 操作
  │
  └─ RoomPage onMounted
       ├─ socket.connect()
       └─ socket.emit('ns:create', { roomId, nickname, playerId, maxPlayers })
               │
               └─ 後端 handler
                    ├─ 寫入 Supabase rooms（service_role，繞過 RLS）
                    ├─ 寫入 Supabase players（第一位玩家 = 房主）
                    ├─ 建立 in-memory cache
                    └─ 推送 room_state 給建立者
```

**優點**：後端使用 service_role，不需要前端有 INSERT 權限；房間建立與加入共用同一套 Socket handler 邏輯；前端 Lobby 完全無 DB 依賴
**缺點**：RoomPage 需在 Socket 連線完成後才能得知建立結果（需處理 loading 狀態）

## 四、統一的前端 Lobby 流程（所有新 App 遵守）

### 4.1 LobbyPage 職責（極簡原則）

```javascript
// ✅ LobbyPage 只做這三件事：
// 1. 輸入驗證（暱稱不為空、房間碼 6 碼）
// 2. 儲存暱稱到 localStorage
// 3. 路由跳轉，把所有資訊用 query 傳給 RoomPage

// 建立房間
function handleCreateRoom() {
  const roomId = generateRoomId()          // shared/data/identity.js
  saveNickname(nickname.value)             // shared/data/identity.js
  router.push({
    name: '[app-slug]-room',
    query: {
      id:     roomId,
      nick:   nickname.value,
      create: '1',           // ← 告知 RoomPage 需要建立
      max:    maxPlayers.value,
    }
  })
}

// 加入房間
function handleJoinRoom() {
  saveNickname(nickname.value)
  router.push({
    name: '[app-slug]-room',
    query: {
      id:   joinCode.value.toUpperCase(),
      nick: nickname.value,
      // 沒有 create=1 → RoomPage 知道是加入而非建立
    }
  })
}
```

> **禁止**：LobbyPage 直接呼叫 Supabase 或 Socket，這些全部交給 RoomPage 的 composable 處理。

### 4.2 RoomPage 初始化邏輯

```javascript
onMounted(() => {
  const roomId     = route.query.id?.toUpperCase()
  const nickname   = route.query.nick || getSavedNickname()
  const isCreating = route.query.create === '1'
  const maxPlayers = parseInt(route.query.max) || 6

  if (!roomId) { router.push({ name: '[app-slug]' }); return }

  if (!nickname) {
    // 顯示暱稱遮罩（直接貼連結進來的情況）
    showNicknameOverlay.value = true
    return
  }

  // 無論建立或加入，都呼叫同一個 connect()
  connect(roomId, nickname, isCreating, maxPlayers)
})
```

### 4.3 Composable connect() 統一入口

```javascript
// useXxxGame.js
function connect(roomId, nickname, isCreating = false, maxPlayers = 6) {
  state.myPlayerId = getOrCreatePlayerId()
  state.myNickname = nickname
  state.roomId     = roomId
  state.loading    = true

  const socket = getSocket()
  if (!socket.connected) socket.connect()

  const doEmit = () => {
    if (isCreating) {
      // 建立：後端負責寫 DB + 建快取
      socket.emit('ns:create', {
        roomId, nickname,
        playerId: state.myPlayerId,
        maxPlayers,
      })
    } else {
      // 加入：後端負責驗證 + 讀 DB
      socket.emit('ns:join', {
        roomId, nickname,
        playerId: state.myPlayerId,
      })
    }
  }

  if (socket.connected) doEmit()
  else socket.once('connect', doEmit)
}
```

## 五、統一的後端 Socket Handler 結構

每個 App 的 namespace 後端都必須實作以下兩個 handler，邏輯保持一致：

```javascript
// server/[app-slug]/index.js

namespace.on('connection', (socket) => {

  // ── 建立房間 ────────────────────────────────────────────────
  socket.on('ns:create', async ({ roomId, nickname, playerId, maxPlayers }) => {
    try {
      // 1. 寫入 Supabase rooms（service_role）
      await db.from('rooms').insert({
        id:             roomId,
        host_player_id: playerId,
        max_players:    maxPlayers,
        app_id:         '[app-slug]',
        status:         'waiting',
      })

      // 2. 寫入 Supabase players（第一位玩家 = 房主）
      await db.from('players').upsert({
        id:         playerId,
        room_id:    roomId,
        nickname,
        is_online:  true,
        join_order: 0,
      }, { onConflict: 'id' })

      // 3. 初始化 in-memory cache
      initCache(roomId, { hostId: playerId, maxPlayers, players: [{ id: playerId, nickname }] })

      socket.join(roomId)
      socket.emit('ns:room-state', buildRoomState(roomId))
    } catch (err) {
      socket.emit('ns:error', { code: 'CREATE_FAILED', message: err.message })
    }
  })

  // ── 加入房間 ────────────────────────────────────────────────
  socket.on('ns:join', async ({ roomId, nickname, playerId }) => {
    // 從 DB 載入（cache miss 時）或從 cache 取
    let room = cache[roomId]?.room
    if (!room) {
      try { room = await db.from('rooms').select('*').eq('id', roomId).single() }
      catch { socket.emit('ns:error', { code: 'ROOM_NOT_FOUND', message: '找不到房間！' }); return }
    }

    // 驗證
    if (cache[roomId]?.players.length >= room.maxPlayers) {
      socket.emit('ns:error', { code: 'ROOM_FULL', message: '房間已滿！' }); return
    }
    if (cache[roomId]?.players.some(p => p.nickname === nickname && p.id !== playerId)) {
      socket.emit('ns:error', { code: 'NICKNAME_TAKEN', message: '此暱稱已被使用！' }); return
    }

    // 寫入 DB + 更新 cache
    // ... （同建立模式的 players upsert）

    socket.join(roomId)
    socket.emit('ns:room-state', buildRoomState(roomId))
    socket.to(roomId).emit('ns:player-joined', { player: { id: playerId, nickname } })
  })

})
```

## 六、分享連結格式規範

所有 App 的分享連結格式統一採用 Hash Router query：

```
https://[host]/#/[app-slug]?room=[ROOMID]
```

範例：
```
https://example.com/#/character-storm?room=AB3K7Z
```

**LobbyPage 貼入邏輯**（`onMounted` 時自動偵測）：

```javascript
onMounted(() => {
  // Hash Router 模式下 query 在 hash 後面
  const hash = window.location.hash          // '#/character-storm?room=AB3K7Z'
  const qs   = hash.includes('?') ? hash.split('?')[1] : ''
  const roomCode = new URLSearchParams(qs).get('room') || ''

  if (roomCode) {
    joinCode.value = roomCode.toUpperCase()
    // 自動 focus 暱稱欄位
    nextTick(() => joinNicknameRef.value?.focus())
    showToast('請輸入暱稱後加入房間 🎉', 'info')
  }
})
```

## 七、新 App 建立房間功能 Checklist

新增一個需要「多人房間」功能的 App 時，按以下清單執行：

### 前端（`vue-app/src/apps/[slug]/`）

- [ ] `pages/[Slug]LobbyPage.vue`
  - [ ] 建立房間卡片（暱稱 + 人數選擇）
  - [ ] 加入房間卡片（房間碼 + 暱稱）
  - [ ] `onMounted` 偵測 `?room=` query，自動填房間碼
  - [ ] `handleCreateRoom`：只呼叫 `generateRoomId()` + `router.push`，**不碰 DB / Socket**
  - [ ] `handleJoinRoom`：只做輸入驗證 + `router.push`，**不碰 DB / Socket**

- [ ] `pages/[Slug]RoomPage.vue`
  - [ ] `onMounted` 讀取 `route.query`（id, nick, create, max）
  - [ ] 呼叫 `composable.connect(roomId, nickname, isCreating, maxPlayers)`
  - [ ] 顯示暱稱遮罩（直接貼連結進來時）

- [ ] `composables/use[Slug].js`
  - [ ] `connect(roomId, nickname, isCreating, maxPlayers)` 統一入口
  - [ ] 連線後依 `isCreating` emit `ns:create` 或 `ns:join`
  - [ ] 處理 `ns:room-state`（加入成功）
  - [ ] 處理 `ns:error`（加入/建立失敗）

### 後端（`server/[slug]/`）

- [ ] `index.js`：`registerNamespace(io)` 建立 namespace
- [ ] 實作 `ns:create` handler（寫 DB + 建 cache + 推 room-state）
- [ ] 實作 `ns:join` handler（驗證 + 讀/寫 DB + 推 room-state）
- [ ] `server.js` 掛載：`require('./[slug]').registerNamespace(io)`

### 資料庫

- [ ] 確認 `rooms` 表有 `app_id` 欄位
- [ ] 後端用 `service_role` key，前端**不**直接呼叫 Supabase 做 INSERT
