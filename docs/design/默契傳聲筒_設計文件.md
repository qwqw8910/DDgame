## 二、遊戲核心流程（Game Flow）

```
大廳等待
  │
  ▼
系統隨機抽題（題目僅提示者可見）
  │
  ▼
【第一輪：殘章】倒數 60 秒，提示者各自輸入（依字數配額）
  │   畫面顯示：每位提示者「已完成 ✓ / 輸入中…」
  │   倒數結束 or 全員送出 → 觸發字元重複判定
  │
  ▼
猜題者看到「符號代換後」的提示 → 第一次答題
  ├── 答對 → 揭曉答案 + 換下一位猜題者
  └── 答錯 → 進入第二輪
              │
              ▼
         【第二輪：長敘】全員固定 4 字，倒數 60 秒
              │   重複判定包含第一輪所有字元（歷史追溯）
              │   相同重複字元維持相同符號
              │
              ▼
         猜題者最終答題
              ├── 答對 → 揭曉答案 + 換下一位猜題者
              └── 答錯 → 揭曉答案 + 換下一位猜題者
                         （第二輪無論對錯皆進入下一輪）
                              │
                              ▼
                    全員輪完一圈（每人都當過猜題者）
                              │
                              ▼
                    結算畫面 → 房主決定「繼續 / 結束」
```

## 三、角色分配邏輯（Role Assignment）

系統根據加入房間的總人數**自動**分配：

| 總人數 | 猜題者 | 2 字提示者 | 4 字提示者 | 6 字提示者 |
|:------:|:------:|:----------:|:----------:|:----------:|
| 4 人   | 1      | 2          | 1          | 0          |
| 5 人   | 1      | 2          | 2          | 0          |
| 6 人   | 1      | 2          | 2          | 1          |
| 7 人   | 1      | 2          | 2          | 2          |
| 8 人以上 | 1    | 3          | 3          | 剩餘全部   |

### 分配規則
- 每局猜題者**輪轉**：第一輪 P1 猜、第二輪 P2 猜……以此類推
- 猜題者**不輸入提示**、**不需知道分配字數**
- 其餘玩家（提示者）由系統依**座位順序**靜態分配字數額度
- 全員輪完一圈（每人都猜過一次）算完成一局

## 四、輸入約束與過濾機制（Filter Rules）

### 4.1 輸入限制（Strict Input）

| 規則 | 說明 |
|------|------|
| **字數強制** | 輸入的中文字數必須**完全等於**分配額度（不含空白），否則按鈕 disabled |
| **黑名單過濾** | 禁止：`0-9` 數字、全/半形標點、英文字母、Emoji |
| **去空白** | 系統自動 trim 首尾空白後再計算字數 |

> 字數計算定義：`[...text.replace(/\s/g, '')].length`（以 Unicode code point 為單位）

### 4.2 字元重複判定與符號代換（核心邏輯）

**判定條件**：字元 $C$ 同時出現在 $\ge 2$ 位**不同玩家**的提示中 → 標記為「重複字元」

**個人內部排除**：玩家 A 輸入「哈哈哈」，若其他玩家皆未輸入「哈」，則「哈」不觸發代換

**符號池（依重複發現順序依序分配）**：

| 順序 | 符號 |
|:----:|:----:|
| 1    | ●    |
| 2    | ▲    |
| 3    | ■    |
| 4    | ◆    |
| 5    | ★    |
| 6    | ✖    |
| 7    | ✚    |
| 8    | ⬢    |

**跨輪一致性（第二輪）**：若字元 $C$ 在第一輪已被映射為符號 $S$，第二輪出現的同一字元 $C$ 必須沿用符號 $S$，協助猜題者建立推理鏈。

## 五、視角呈現（View Management）

### 5.1 提示者視角

- 顯示自己輸入的**原文字**
- 若某字被判定為重複，在該字下方標註 `❌` 或文字變紅
- 提示已送出的其他提示者顯示「✓ 已完成」（不顯示內容）
- 提示尚未送出的提示者顯示「輸入中…」

### 5.2 猜題者視角

- 僅能看到**符號代換後**的提示
- 無法看到任何提示者的原文

### 5.3 顯示範例

```
題目：「臭豆腐」（猜題者不可見）

提示 A（2字）：發酵        → 猜題者看到：發酵
提示 B（2字）：好●        → 猜題者看到：好●   （● = 臭）
提示 C（4字）：聞起來●    → 猜題者看到：聞起來●
提示 D（4字）：台灣▲吃    → 猜題者看到：台灣▲吃 （▲ = 豆）

進入第二輪後，若再出現「臭」仍顯示 ●
```

## 六、題庫設計（Supabase）

### 6.1 資料表：`character_storm_words`

```sql
CREATE TABLE character_storm_words (
  id         BIGSERIAL PRIMARY KEY,
  word       TEXT        NOT NULL,        -- 題目（2~4個中文字為主）
  category   TEXT        NOT NULL,        -- 分類，例如：食物、地名、人物
  difficulty TEXT        NOT NULL DEFAULT 'normal', -- easy / normal / hard
  is_active  BOOLEAN     NOT NULL DEFAULT TRUE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### 6.2 抽題邏輯（後端）

```javascript
// 從資料庫隨機抽一題（排除此局已用過的）
async function drawWord(usedIds = []) {
  const { data, error } = await db
    .from('character_storm_words')
    .select('id, word, category')
    .eq('is_active', true)
    .not('id', 'in', `(${usedIds.join(',') || 0})`)
    .order('random()')  // Supabase 支援
    .limit(1)
    .single();
  if (error) throw error;
  return data;
}
```

## 八、核心演算法（filterLogic.js）

```javascript
/**
 * 字元重複判定與符號代換
 * @param {Array<{id: string, text: string}>} hints - 本輪提示
 * @param {Object} inheritedSymbolMap - 上一輪已建立的映射（第二輪傳入，第一輪傳 {}）
 * @returns {{ forProvider, forGuesser, symbolMap }}
 */
function processHints(hints, inheritedSymbolMap = {}) {
  const SYMBOLS = ['●', '▲', '■', '◆', '★', '✖', '✚', '⬢'];
  const charPlayerMap = {}; // { '字': Set(playerId) }

  // Step 1：統計每個字出現在幾位玩家的提示中
  hints.forEach(({ id, text }) => {
    [...text].forEach(char => {
      if (!charPlayerMap[char]) charPlayerMap[char] = new Set();
      charPlayerMap[char].add(id);
    });
  });

  // Step 2：建立 symbolMap（繼承上輪 + 補充新發現的重複字）
  const symbolMap = { ...inheritedSymbolMap };
  const usedSymbols = new Set(Object.values(symbolMap));
  let symbolPool = SYMBOLS.filter(s => !usedSymbols.has(s));

  Object.entries(charPlayerMap).forEach(([char, playerSet]) => {
    if (playerSet.size >= 2 && !symbolMap[char]) {
      symbolMap[char] = symbolPool.shift() ?? '?';
    }
  });

  // Step 3：產出雙視角結果
  const forProvider = hints.map(({ id, text }) => ({
    playerId: id,
    chars: [...text].map(char => ({
      char,
      isConflict: !!symbolMap[char],
    })),
  }));

  const forGuesser = hints.map(({ id, text }) => ({
    playerId: id,
    maskedText: [...text].map(c => symbolMap[c] ?? c).join(''),
  }));

  return { forProvider, forGuesser, symbolMap };
}

module.exports = { processHints };
```

### 9.2 兩頁架構（與「懂我再說」相同模式）

| 路由 | 元件 | 說明 |
|------|------|------|
| `/character-storm` | `CharacterStormLobbyPage.vue` | 建立/加入房間，分享邀請連結 |
| `/character-storm/room` | `CharacterStormRoomPage.vue` | 遊戲主畫面，所有狀態切換 |

**建立房間流程**：
1. 玩家在 LobbyPage 輸入暱稱、選最大人數 → 點「建立房間」
2. 前端 `generateRoomId()` 產生 6 碼房間碼
3. 路由跳轉到 `/character-storm/room?id=XXXXXX&nickname=YYY&create=1&max=N`
4. RoomPage `onMounted` 讀取 query params，emit `cs:create` 到 Socket namespace

**加入房間流程**：
1. 玩家輸入 6 碼房間碼 + 暱稱 → 點「加入房間」
2. 路由跳轉到 `/character-storm/room?id=XXXXXX&nickname=YYY`
3. RoomPage `onMounted` emit `cs:join` 到 Socket namespace

**分享連結格式**：
```
https://[host]/#/character-storm?room=XXXXXX
```
- 對方點連結 → LobbyPage 自動填入房間碼 → 輸入暱稱後加入

## 十、頁面 UI 狀態規格

### 10.1 大廳（waiting）

- 房間碼 + 一鍵複製按鈕
- 玩家列表（暱稱、連線狀態、預計角色）
- 「開始遊戲」按鈕（僅房主，人數 ≥ 4 才啟用）
- 非房主顯示「等待房主開始…」

### 10.2 第一輪提示輸入（round1）

**提示者畫面**
- 顯示題目（例如：「臭豆腐」）
- 分配字數說明（「你是 2 字提示者」）
- 文字輸入框（即時字數計算，超過或不足則鎖定送出）
- 送出按鈕（字數完全符合才啟用）
- 玩家進度列表（「✓ 小明已完成 ／ 阿花輸入中…」）
- 倒數計時器（CSTimer.vue）

**猜題者畫面**
- 顯示「等待提示者輸入中…」
- 玩家進度列表（同上，只看狀態不看內容）
- 倒數計時器

### 10.3 第一輪猜題（round1-result）

**猜題者畫面**
- 顯示所有提示（符號代換後）
- 文字輸入框：自由輸入答案
- 「送出答案」按鈕

**提示者畫面**
- 顯示自己提示的原文 + 衝突標記（紅字或 ❌）
- 顯示「等待猜題者答題中…」

### 10.4 揭曉（revealing）

- 大字顯示正確答案
- 猜題者是否猜對（✓ / ✗）
- 第幾輪答對（第一輪 / 第二輪 / 未猜出）
- 「下一位」按鈕（僅房主）

### 10.5 結算（finished）

- 本局結束，顯示所有玩家的猜題結果摘要
- 「繼續下一局」按鈕（房主）
- 「結束遊戲」按鈕（房主）

## 十一、錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 玩家在輸入途中斷線 | 倒數到期後跳過該玩家（視為空白），遊戲繼續 |
| 房主斷線 | 轉移房主身份給下一位已連線玩家 |
| 倒數結束但仍有人未送出 | 未送出者的提示視為空字串，不參與重複判定 |
| 符號池耗盡（>8 組重複） | 超出部分以 `?` 代替，log warning |
| 題庫已用完（usedWordIds 超過總數） | 清空 usedWordIds，從頭再抽 |
